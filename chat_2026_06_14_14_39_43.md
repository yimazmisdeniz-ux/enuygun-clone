Sitenizin "güvenli gözükmüyor" uyarısı vermesi genellikle SSL/TLS sertifikası ile ilgili bir problem olduğunu gösterir. Bu sorunu çözmek için aşağıdaki adımları takip edebilirsiniz:

### 1. SSL Sertifikası Kontrolü
- Sitenizin geçerli bir SSL sertifikasına sahip olduğundan emin olun.
- Ücretsiz sertifika için Let's Encrypt kullanabilirsiniz.
- ```openssl s_client -connect siteniz.com:443``` komutuyla sertifika durumunu kontrol edebilirsiniz.

### 2. HTTPS Zorunlu Kılın
Tüm HTTP trafiğini HTTPS'e yönlendirin:

**Apache için (.htaccess):**
```apache
RewriteEngine On
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]
```

**Nginx için:**
```nginx
server {
    listen 80;
    server_name siteniz.com;
    return 301 https://$host$request_uri;
}
```

### 3. Mixed Content Sorunları
- HTTPS sayfalarında HTTP ile yüklenen kaynaklar (resim, script, CSS) olup olmadığını kontrol edin.
- Browser konsolunda "Mixed Content" uyarılarını inceleyin.

### 4. Telegram API Kullanımı (Eğer varsa)
TG (Telegram) botu kullanıyorsanız ve veri aktarımı yapıyorsanız:
- API endpointlerinizin HTTPS üzerinden çalıştığından emin olun
- Telegram Bot API'si zaten HTTPS kullanır, ek bir önlem gerekmez

### 5. Sertifika Zinciri Sorunları
- Ara sertifikaların eksik olup olmadığını kontrol edin:
```bash
curl -Iv https://siteniz.com 2>&1 | grep "SSL certificate"
```

### 6. HSTS Header Ekleme
```nginx
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
```

### 7. Son Kontroller
- **[SSL Labs Test](https://www.ssllabs.com/ssltest/)** ile detaylı kontrol yapın
- Sertifikanın süresinin dolmadığından emin olun

Bu adımları uyguladıktan sonra tarayıcı çubuğunda "Güvenli" ifadesini görmelisiniz. Sorun devam ederse hangi web sunucusu ve hosting ortamı kullandığınızı bildirerek daha spesifik çözümler sunabilirim.
